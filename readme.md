# 📧 Mail Robot: Автоматизированный Проверщик Почты

## 📖 Описание

`check_mail.py` — это инструмент для автоматической проверки нескольких почтовых аккаунтов на наличие новых сообщений с использованием протокола IMAP.

### Что делает скрипт:

1. **Подключается** к почтовому серверу через IMAP (поддерживает SSL и STARTTLS)
2. **Анализирует** заголовки всех сообщений в почтовом ящике
3. **Извлекает** тему письма, отправителя и дату
4. **Логгирует** все действия в консоль и в файл `mail_check.log`
5. **Сохраняет** данные о последней проверке в SQLite-базе данных

---

## 🚀 Возможности

*   **Поддержка нескольких аккаунтов:** Настройте любое количество почтовых ящиков в файле `accounts.json`
*   **IMAP Протокол:** Прямое подключение к почтовому серверу с поддержкой SSL
*   **Полный скан:** Проверяет весь почтовый ящик (не ограничивается по дате)
*   **Профессиональное логирование:** Вывод в консоль и в файл `mail_check.log`
*   **Безопасность:** Поддержка хранения паролей через переменные окружения
*   **Универсальность:** Работает с любыми IMAP-совместимыми серверами

### Конфигурация (`accounts.json`)

Просто укажите данные аккаунтов в файле `accounts.json`:

```json
[
    {
        "email": "USER_EMAIL_HERE",
        "password": "USER_PASSWORD_HERE",
        "imap_server": "USER_IMAP_SERVER_HERE",
        "imap_port": 993,
        "imap_security": "ssl"
    }
]
```

**📋 Описание полей:**

| Поле | Описание |
|------|---------|
| `email` | Полный адрес электронной почты |
| `password` | Пароль от аккаунта |
| `imap_server` | Хост почтового сервера (например, `imap.beget.com`) |
| `imap_port` | Порт соединения (обычно 993 для SSL) |
| `imap_security` | Тип шифрования: `ssl` или `starttls` |
| `start_date` | (Опционально) Дата начала для фильтрации по дате |
| `end_date` | (Опционально) Дата окончания для фильтрации по дате |

## 🔄 Принцип работы (логика)

1.  **Загрузка аккаунтов:** Чтение конфигурации из `accounts.json`
2.  **Подключение к IMAP:** Установление соединения с сервером (SSL/STARTTLS)
3.  **Вход в аккаунт:** Аутентификация с использованием пароля
4.  **Полный скан:** Поисковый запрос `ALL` для получения всех сообщений
5.  **Парсинг заголовков:** Извлечение темы, отправителя и даты
6.  **Логирование:** Запись всех событий в консоль и файл
7.  **Завершение:** Вывод сводки по всем аккаунтам

## ⚠️ Типичные ошибки и решения

### 1. Ошибка подключения к серверу

```
INFO: --- Connecting to USER_IMAP_SERVER_HERE for USER_EMAIL_HERE ---
INFO: Attempting to log in...
INFO: [WARNING] Authentication or Connection Error: ...
```

**Решение:**
*   Проверьте, что сервер указан правильно (`imap_server`)
*   Убедитесь, что порт корректный (`imap_port`)
*   Проверьте тип шифрования (`imap_security` — `ssl` или `starttls`)
*   Проверьте, что сервер поддерживает IMAP

### 2. Ошибка авторизации

```
INFO: Successfully logged in.
INFO: [INFO] Found 0 total messages in the mailbox.
```

**Решение:**
*   Проверьте, что пароль указан верно
*   Проверьте, что аккаунт активен и не заблокирован

### 3. Ошибка чтения пароля из переменной окружения

```
INFO: Raw password from config: 'USER_PASSWORD_HERE'
INFO: Resolved password being used: 'YOUR_PASSWORD_HERE'
```

**Решение:**
*   Убедитесь, что переменная окружения установлена
*   Проверьте, что имя переменной совпадает (регистр не важен)

## 📝 Примечания

### Поддерживаемые IMAP-серверы

*   **Gmail:** `imap.gmail.com` (порт 993, SSL)
*   **Outlook/Hotmail:** `outlook.office365.com` (порт 993, SSL)
*   **Yandex:** `imap.mail.yandex.ru` (порт 993, SSL)
*   **Beget:** `imap.beget.com` (порт 993, SSL) — *пример сервера*
*   **Beget STARTTLS:** `imap.beget.com` (порт 143, STARTTLS)

### Настройка IMAP для Gmail

1.  Зайдите в настройки аккаунта Gmail
2.  В разделе "Безопасность" включите **"Двухфакторная аутентификация"**
3.  Включите **"Доступ приложений"** (App Passwords)
4.  Используйте сгенерированный пароль приложений вместо основного пароля

### Настройка IMAP для Yandex

1.  Зайдите в настройки аккаунта Yandex
2.  Включите **"IMAP"** в разделе "Подключения к другим сервисам"
3.  Используйте `imap.mail.yandex.ru` (порт 993, SSL)
